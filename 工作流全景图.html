循环迭代研发 Agent 团队

从需求到交付的完整闭环——六角色协作、五阶段循环、零业务中断的安全迭代体系

六角色团队

🏗

产品架构师

Product Architect
兼具顶级产品思维与底层架构把控力，将业务需求转化为演进式架构蓝图，确保快速验证业务闭环同时不留技术死角。
核心输出
演进式架构落地切片计划
🛡

安全切片规划师

Safe Slicing Planner
将任何复杂变更转化为零业务中断、可安全回滚的工程化执行步骤，是团队的"安全阀"。
核心输出
安全执行切片计划
🔍

代码审查专家

Code Review Expert
15年+经验的"找茬"专家，四维度深度审查：功能性Bug、安全漏洞、代码质量、业务逻辑，一个不漏。
核心输出
四维度审查报告
⚖

架构审计师

Architecture Auditor
宏观层面的"体检医生"，评估架构合理性、业务闭环、项目完成度，识别架构债务和系统性风险。
核心输出
架构审计报告
🔧

自动修复工程师

Auto Fix Engineer
按切片计划逐项核实并修复问题，遵循"最小改动、最大安全"原则，先核实再动手，不引入新依赖。
核心输出
修复摘要报告
📋

运维管家

DevOps Steward
团队的"大管家"，负责垃圾文件清理、工作记录、迭代节奏管理和文档归档，确保上下文不丢失。
核心输出
工作记录 / 迭代总结

五阶段工作流

0

迭代启动

运维管家 准备阶段
读取上一轮迭代总结恢复上下文，确认本轮迭代目标和范围，准备输入材料。
输入
上一轮迭代总结、需求文档
输出
迭代启动声明
1

架构规划

产品架构师 安全切片规划师
产品架构师输出演进式架构落地切片计划（MVP边界、API契约、防腐层），安全切片规划师据此制定安全执行切片计划（依赖拓扑、兼容策略、回滚方案）。
门控标准
架构计划 + 切片计划均确认通过
核心风险
过早过度设计 / 边界模糊
2

代码实现

开发团队 受切片计划约束
按安全执行切片计划的步骤顺序实现代码，每个步骤完成后执行验收标准，验收通过后进入下一步，失败则回滚重做。
约束
严格按步骤顺序，不得跳步
输出
实现代码 + 各步骤验收记录
3

代码审查

代码审查专家 架构审计师
代码审查专家进行四维度深度审查（功能Bug→安全→质量→业务逻辑），架构审计师进行宏观审计（架构合理性→业务闭环→完成度→缺陷排查）。
决策树
P0→立即修复 / P1→修复 / P2→评估 / P3→推迟
输出
审查报告 + 审计报告
4

安全修复

安全切片规划师 自动修复工程师
安全切片规划师制定修复切片计划，自动修复工程师按四阶段执行（快速修复→需测试验证→仅评估→跳过），修复后回归审查。
回归审查
通过→收尾 / 新P0P1→重新规划 / 未解决→重新修复
输出
修复摘要报告 + 验证结果
5

迭代收尾

运维管家
垃圾文件清理（确认后执行）、工作记录、迭代总结、文档归档。准备下一轮迭代的输入。
完成标准
构建通过 + 测试通过 + 文档归档
输出
迭代总结 + 遗留问题清单

循环迭代机制

持续迭代，直到交付

每一轮迭代都是"规划→实现→审查→修复→收尾"的完整闭环，遗留问题自动流入下一轮

审查结果决策树

审查完成，是否存在问题？

代码审查专家 + 架构审计师联合输出

P0 致命问题 → 立即修复

崩溃、数据丢失、安全漏洞 → 进入阶段四安全修复

P1 高优先级 → 本轮修复

大概率触发Bug、明显安全漏洞 → 进入阶段四安全修复

P2 中优先级 → 评估决定

逻辑不严谨、一致性隐患 → 评估是否本轮修复或推迟到下一迭代

P3 低优先级 → 推迟处理

代码规范、命名、可读性 → 记录，推迟到后续迭代

无问题 → 进入收尾

审查通过，进入阶段五迭代收尾

质量门控

1

架构规划门控

架构计划 + 切片计划均确认通过

2

代码实现门控

所有步骤验收标准通过

3

代码审查门控

无 P0 级致命问题

4

安全修复门控

回归审查通过，无新问题

5

迭代收尾门控

构建通过 + 测试通过 + 文档归档

核心原则

🚫

禁止直接生成最终代码

架构师和规划师只输出计划，不输出代码实现

📜

契约先行

先定义 API 契约，再实现业务逻辑

🛡

防腐层设计

破坏性修改必须有兼容层，旧逻辑先透传再剥离

🔬

最小改动

修复遵循最小改动原则，不引入新依赖

✅

先核实再动手

修复前先确认问题是否真实存在，避免误修复

🔄

可回滚性

每个步骤必须可独立回滚，避免状态不一致

🎯

端到端验收

验收标准是业务场景验证，不是"代码不报错"

🔁

循环迭代

问题未完全解决则自动流入下一轮迭代
